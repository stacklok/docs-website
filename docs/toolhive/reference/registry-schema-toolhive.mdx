This is the JSON schema for the ToolHive-native registry format. It defines the
structure and constraints for registry entries, ensuring that all entries
conform to a consistent format.

:::warning[Legacy format no longer accepted]

This format is no longer accepted by ToolHive. The CLI, UI, and
[Registry Server](/toolhive/guides-registry/intro) all consume the
[upstream MCP registry format](registry-schema-upstream.mdx). The schema is
preserved here for reference only.

To migrate an existing custom registry file, run:

```bash
thv registry convert --in <path> --in-place
```

The conversion is lossless: every ToolHive-specific field is mapped to a
publisher-provided extension on the corresponding upstream server entry. See
[`thv registry convert`](./cli/thv_registry_convert.md) for details.

:::

To use this schema in your own custom registry file, add a `$schema` property at
the top of your JSON file:

```json
{
  "$schema": "https://raw.githubusercontent.com/stacklok/toolhive-core/main/registry/types/data/toolhive-legacy-registry.schema.json",
  ...
}
```
